# Hair Salon Simulator — Spec

## 1. コンセプト

「ストランドレベルの髪物理演算」×「美容師シミュレーション」を組み合わせた、
まだ市場に存在しないカテゴリのゲーム。

**ターゲット**: ゲーマー、美容師志望者、カジュアルシミュレーション好き
**差別化**: 既存タイトル（Hairdresser Simulator等）はセクション分割式の簡易表現。
本作はストランド単位の物理演算で「1本1本切れる」リアルさを実現する。

---

## 2. Tech Stack

| 項目 | 選定 | 理由 |
|---|---|---|
| Engine | Unity 6 (URP) | com.unity.demoteam.hair が公式サポート |
| Hair Physics | com.unity.demoteam.hair | GPU駆動、数万本ストランド、無料・公式 |
| Rendering | URP + カスタムヘアシェーダー | パフォーマンスとビジュアルのバランス |
| Input | Mouse + Keyboard（Phase 1） | VRは将来フェーズ |
| Platform | PC (Windows) | GPU性能が必要なためPC優先 |

## 3. デモ（Phase 1）のスコープ

### 3.1 ゴール

**「マネキンヘッドの髪をハサミで切れるデモ」** を作る。
ゲーム要素は後から足す。まず物理演算＋カット操作の手触りを確認する。

### 3.2 必要な要素

#### A. マネキンヘッド
- 静的な頭部3Dモデル（アセットストアまたは無料モデル）
- 頭皮メッシュにヘアルートをマッピング

#### B. ヘアシステム
- com.unity.demoteam.hair をセットアップ
- ストランド数: 5,000〜10,000本（デモ用、パフォーマンス検証）
- 髪の長さ: ミディアム〜ロング（カットの変化が分かりやすい）
- 重力・風の基本物理
- ストランド同士の衝突（基本レベル）

#### C. ハサミツール
- マウスカーソル位置にハサミを表示
- クリック or ドラッグでカット操作
- カット判定: レイキャスト → ストランドとの交差検出 → 交差点でストランドを分割
- カットされた髪は短くなり、切り落とされた部分は落下（重力）

#### D. カメラ操作
- マネキンヘッドを中心にオービットカメラ
- ズームイン/アウト
- 右ドラッグで回転

#### E. UI（最小限）
- ツール選択（Phase 1ではハサミのみ）
- リセットボタン（髪を元に戻す）

### 3.3 スコープ外（Phase 1では作らない）
- サロン環境・内装
- 顧客AI・経営要素
- ドライヤー・カラー・パーマ等の追加ツール
- VR対応
- サウンド
- セーブ/ロード

## 5. リスクと対策

| リスク | 影響 | 対策 |
|---|---|---|
| com.unity.demoteam.hair がカット操作を想定していない | ストランド分割が困難 | ソースコードを読んで拡張する。最悪、ストランドの長さパラメータを動的に変更する方式で代替 |
| GPU負荷が高すぎる | フレームレート低下 | ストランド数を減らす、LOD活用、ガイドヘア＋フォローヘア方式で間引く |
| 頭部モデルとヘアルートの整合性 | 髪が頭を貫通する等 | SDF（Signed Distance Field）ベースの衝突で対処 |
